# TC-11 局内战斗场景测试

> 本文件为**战斗专项场景测试**，聚焦局内战斗系统的协同表现。
> 使用控制台快速搭建特定战斗场景，验证各战斗系统在实际运行中的交互结果。
> 与 TC-02/TC-03/TC-04 的模块单元测试互补，侧重**多系统联动**和**边界条件**。

---

## 场景 S01：武器羁绊 + 技能联动（综合输出验证）

**场景描述**：触发速射T2羁绊，同时使用超载技能，验证多重加成叠加是否正常。

**搭建步骤**：
1. 控制台依次：`weapon smg{ENTER}` → `weapon gatling{ENTER}` → `weapon burst_pistol{ENTER}`
2. 控制台：`skill overload{ENTER}`
3. 关闭控制台，等待 1000ms，截图

**预期**：
- 速射T2羁绊激活（移速+50，伤害+8）
- 发送 `f` 开启超载后武器射速再次翻倍
- 三把武器同时开火，子弹极度密集
- 击杀速度明显快于无加成状态

**截图时机**：超载激活后 500ms

## 场景 S02：满背包融合场景

**场景描述**：背包装满后触发融合，验证融合后背包空间释放正常。

**搭建步骤**：
1. 控制台：`weapon pistol{ENTER}` → `weapon smg{ENTER}`（可触发融合）
2. 打开背包（TAB），将 smg 拖到 pistol 位置
3. 截图（融合预览）
4. 确认融合（ENTER）
5. 截图（融合后背包状态）

**预期**：
- 融合预览界面显示 dual_pistol 属性
- 融合后背包中 pistol + smg 消失，出现 dual_pistol
- 背包空出原来两把武器的格子（空间正确释放）
- dual_pistol 有橙色/特殊颜色边框标记

## 场景 S03：低血量生存极限

**场景描述**：模拟濒死状态，验证无敌帧、护盾等保命机制。

**搭建步骤**：
1. 控制台：`hp 5{ENTER}` 设置极低血量
2. 控制台：`skill mana_shield{ENTER}` 获取护盾技能
3. 发送 `f` 激活护盾
4. 等待敌人攻击（约 3000ms）
5. 截图

**预期**：
- 护盾激活时 Buff HUD 显示护盾图标
- 受到一次伤害后护盾消失（图标消失），血量不减少
- 护盾消失后再次受伤，血量正常扣除
- 血量到 0 后进入结算（不无限护盾循环）

## 场景 S04：经验爆发升级连锁

**场景描述**：一次性大量获取经验，验证连续升级和奖励选择队列。

**搭建步骤**：
1. 控制台：`exp 9999{ENTER}`
2. 关闭控制台，等待 500ms，截图

**预期**：
- 连续弹出多个升级奖励界面（不叠加，依次弹出）
- 每次选择后立即弹出下一个

## 场景 S05：技能冷却 & 多技能队列

**场景描述**：快速连按多个技能键，验证 CD 控制和技能不乱序触发。

**搭建步骤**：
1. 控制台：`skill dash{ENTER}` → `skill overload{ENTER}` → `skill battle_cry{ENTER}`
2. 关闭控制台
3. 快速依次发送 `{SPACE}` → `f` → `q`（间隔 100ms）
4. 等待 1000ms，截图

**预期**：
- 三个技能按顺序各自独立触发
- 各技能 CD 条独立倒计时
- CD 中再按对应键无效（不重复触发）
- HUD 技能槽显示各槽倒计时状态

## 场景 S06：同槽技能替换流程

**场景描述**：F槽已有技能时选择新F槽技能，验证替换确认流程。

**搭建步骤**：
1. 控制台：`skill overload{ENTER}`（工程师F槽 = overload）
2. 触发升级界面（`level 5{ENTER}`），选到 battle_cry 奖励
3. 截图（是否弹出替换确认）
4. 确认替换（ENTER）
5. 截图（F槽技能是否变化）

**预期**：
- 弹出"是否替换F槽技能"确认界面
- 确认后 F 槽变为 battle_cry
- 原 overload 技能消失
- 发送 `f` 验证新技能生效

## 场景 S07：灵魂刷新升级奖励

**场景描述**：在升级界面使用灵魂刷新奖励池，验证刷新机制。

**搭建步骤**：
1. 控制台：`souls 500{ENTER}` 设置足量灵魂
2. 控制台：`level 2{ENTER}` 触发升级
3. 等待升级界面弹出，截图（刷新前奖励）
4. 找到并使用刷新操作（对应按键）
5. 截图（刷新后奖励）

**预期**：
- 灵魂数减少（消耗固定数量）
- 3张奖励卡片全部变为新内容
- 可多次刷新（每次消耗灵魂递增或固定）

## 场景 S08：里程碑达成验证

**场景描述**：达成角色里程碑条件，验证里程碑触发和点数发放。

**搭建步骤**：
1. 选择工程师开始游戏
2. 控制台 `skill overload{ENTER}`，多次使用 `f` 触发超载（达成"使用超载X次"里程碑）
3. 截图（是否有里程碑达成提示）
4. 游戏结算后截图（里程碑点数是否增加）

**预期**：
- 游戏内出现里程碑达成浮动提示
- 结算界面显示里程碑点数增加
- 进入成长界面，工程师技能树里程碑点数增加

## 场景 S09：长局耐久性（5分钟）

**场景描述**：持续存活5分钟，验证游戏无内存泄漏/性能下降/崩溃。

**搭建步骤**：
1. 正常开始游戏
2. 控制台：`set speed 400{ENTER}`（方便存活）
3. 每 60 秒截图一次，共截图 5 次

**预期**：
- 5分钟内游戏不崩溃
- 帧率保持稳定（截图中动画流畅）
- 敌人数量随时间增多但不卡顿
- 经验、灵魂、升级均正常累积

## 场景 S10：死亡复活流程（若有复活机制）

**场景描述**：验证复活功能（Phase 10 引入的 revives 机制）。

**搭建步骤**：
1. 确认玩家有复活次数（初始 _revives = 1）
2. 控制台 `hp 1{ENTER}`，等待死亡
3. 截图（复活界面或直接复活效果）

**预期**：
- 死亡后若有复活次数，触发复活流程
- 复活后有无敌帧（Buff HUD 显示无敌图标）
- 复活次数减少1
- 第二次死亡（无复活次数）直接进入结算
