# Zero 游戏功能测试用例

> 本目录为**游戏端到端功能测试用例**，通过 `/autotest` 自动化执行。
> 与 `tests/` 下的单元测试（Lua/lupa）互补，覆盖实际游戏运行行为。

## 目录结构

```
tests/gameplay/
├── README.md                    — 本文件，索引与规范
├── TC-00-full-flow.md           — ★ 完整游戏流程（必跑，每轮第一个）
├── TC-01-menu.md                — 主菜单 & 角色选择（9条）
├── TC-02-combat.md              — 战斗核心（9条）
├── TC-03-weapons.md             — 武器系统（15条）
├── TC-04-skills.md              — 技能系统（9条）
├── TC-05-bag.md                 — 背包系统（13条）
├── TC-06-synergy.md             — Tag 羁绊系统（8条）
├── TC-07-upgrade.md             — 升级奖励选择（8条）
├── TC-08-progression.md         — 局外成长界面（15条）
├── TC-09-gameover.md            — 游戏结算流程（8条）
├── TC-10-console.md             — 开发者控制台（16条）
└── TC-11-combat-scenario.md     — ★ 局内战斗场景（10个场景）
```

## 执行顺序

1. **TC-00（完整流程）**：每轮必跑，验证主线无断点
2. **TC-01 ~ TC-10（模块单元）**：按编号顺序逐模块执行
3. **TC-11（战斗场景）**：最后执行，验证多系统联动

## 用例状态说明

| 标记 | 含义 |
|------|------|
| ✅ | 通过 |
| ❌ | 失败（需修复） |
| ⚠️ | 异常（结果不确定，需人工确认） |
| ⏭ | 跳过（前置条件未满足） |

## 用例维护规则

- **新增功能** → 在对应 TC 文件末尾追加用例（编号顺延）
- **功能迭代** → 修改对应用例的操作/预期描述
- **功能删除** → 对应用例标注 `> ⛔ 已删除（PhaseX）` 或直接删除
- 每次 Phase 结束后由 `/update-testcases` skill 自动同步
