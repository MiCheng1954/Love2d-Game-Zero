# 2026-03-24 — 0.11 版本收尾会议：测试效率提升

## 会议背景

Zero 项目 0.11 版本全部 13 个 Phase 开发完成。本次会议讨论如何提升后续开发的测试效率。

---

## 讨论摘要

当前最大瓶颈：UwU~ 无法直接进入游戏测试，所有 UI 验证和功能验证都依赖冰冰手动操作后反馈，导致 Bug 定位慢、修复验证慢。

会议确定了两个核心改进方向：

1. **截图能力**：UwU~ 可以截取游戏窗口画面，主动看到当前 UI 状态
2. **自动化测试体系**：通过 PowerShell 发送按键模拟玩家操作，结合截图验证，实现端到端自动测试

---

## 问答记录

**Q: 你有什么办法解决测试效率问题？**

UwU~ 的方案：
- 截图（PowerShell 窗口截图 → Read 工具分析）
- SendKeys（PowerShell 向游戏窗口发送按键，模拟操作）
- 组合使用：操作 → 截图 → 分析，形成验证闭环

**Q: 截图能只截游戏窗口而不是全屏吗？**

可以。通过 `GetWindowRect` Win32 API 找到 Love2D 进程的窗口句柄，只截取该窗口区域的像素。已实现并测试通过。

**Q: 希望能自动化跑主要流程，针对各模块输出测试用例，然后自己测试验证**

已确定 7 个测试模块，每个模块包含若干用例（共约 20 个）：

| 模块 | 说明 | 主要用例 |
|------|------|---------|
| A — 主菜单 | 菜单显示和导航 | A-1 菜单显示, A-2 导航 |
| B — 角色选择 | 角色切换和开始游戏 | B-1 进入, B-2 切换, B-3 确认 |
| C — 武器测试 | 各武器添加和射击效果 | C-1~C-4 手枪/霰弹/狙击等 |
| D — 技能测试 | 升级奖励 + 主动技能 | D-1 强制升级, D-2 选奖励, D-3 用技能 |
| E — 背包系统 | 打开/浏览/关闭背包 | E-1~E-3 |
| F — 局外成长 | 两个 Tab + 英雄切换 | F-1~F-5 |
| G — 游戏结算 | 触发死亡进入结算 | G-1 |

**Q: 将自动化测试作为 skill 保留下来，定期安排测试**

已创建 `/autotest` slash command，保存在 `.claude/commands/autotest.md`。

---

## 最终决策

| 决策 | 内容 |
|------|------|
| 截图工具 | `.claude/screenshot.ps1`，`/screenshot` 随时调用 |
| 按键发送工具 | `.claude/sendkey.ps1`，支持单键 / 文字输入 / 特殊键 |
| 自动化测试 skill | `/autotest`，7 模块约 20 用例，执行后输出结构化报告 |
| 测试触发方式 | 由冰冰定期手动触发 `/autotest`，或在 Phase 结束后执行 |
| Bug 记录方式 | 测试中发现的问题直接写入 `data/bugs.json` |
