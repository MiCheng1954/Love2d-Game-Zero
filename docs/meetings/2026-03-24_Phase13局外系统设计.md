# QA 会议记录 — Phase 13 局外系统设计

**日期：** 2026-03-24

**议题：** Phase 13 局外系统核心设计决策

---

## 完整问答记录

### Q1 — 局外成长模式
**问：** 倾向于轻量加成型 / 解锁树型 / 双轨制？
**答：C — 双轨制**，永久加成（通用）+ 角色/武器解锁，两套独立互不干扰。

---

### Q2 — 角色系统范围
**问：** 实现几个角色？差异化深度如何？
**答：** 先做 **3 个角色**，每个角色有专属技能 + 专属升级选项池 + **角色专属局外成长树**。

---

### Q3 — 3 个角色定位
**问：** 另外两个角色的风格方向？
**答：** 选 **A（狂战士）+ C（幽灵）**，同时将现有 `default` 改名为**工程师**，最终 3 个英雄：

| 角色 | 风格 | 专属技能方向 |
|------|------|------------|
| 工程师（原 default） | 科技 / 超载 | overload（武器射速翻倍） |
| 狂战士 | 近身爆发 / 高风险高回报 | 血量越低攻击力越高、冲刺反伤 |
| 幽灵 | 速度 / 闪避 | 瞬移强化、减速领域扩大 |

---

### Q4 — 角色专属局外成长结构
**问：** 角色专属成长是共享货币独立树 / 专属货币 / 里程碑解锁？
**答：C — 里程碑解锁**，达成与角色特性相关的里程碑目标，获得该角色的里程碑奖励点数，用点数解锁角色专属技能树节点。

---

### Q5 — 角色技能树形态
**问：** 线性升级 / 分支树 / 星盘型？
**答：B — 分支树**，有若干主干，主干下有分支，解锁前置节点才能点后续。

---

### Q6 — 通用永久加成轨结构
**问：** 轻量扁平 / 分3档升级 / 结算奖励驱动？
**答：C — 结算奖励驱动**，每局结算按表现（击杀数/存活时间/Boss击败数）获得成长点数，累积后花在通用加成上。

---

### Q7 — 角色解锁条件
**问：** 通用点数阈值 / 特定里程碑 / 二者结合？
**答：** 暂时**不设置解锁门槛**，现阶段所有角色直接可选，解锁机制后续再设计。

---

### Q8 — 成就系统范围
**问：** Phase 13 一起做 / 只做里程碑 / 搭框架不填内容？
**答：C — 搭框架**，建好成就管理器和 UI 壳子，成就条目内容后续快速添加，Phase 13 不填充具体成就。

---

## 最终决策表

| 模块 | 决策 |
|------|------|
| **局外成长模式** | 双轨制：通用成长轨（结算驱动）+ 角色专属成长轨（里程碑驱动） |
| **角色数量** | 3 个：工程师 / 狂战士 / 幽灵 |
| **default → 工程师** | `characterId = "default"` 改为 `"engineer"`，显示名更新 |
| **角色差异化** | 每角色：基础属性差异 + 专属技能(F槽) + 专属升级选项池 + 专属分支技能树 |
| **里程碑系统** | 与角色特性绑定的条件目标 → 积累点数 → 解锁专属分支树节点 |
| **技能树形态** | 分支树（主干 + 分支，前置解锁后续） |
| **通用加成轨** | 结算按表现给点数（击杀/存活/Boss），累积花在通用属性加成上 |
| **角色解锁** | 暂不设门槛，所有角色直接可选 |
| **成就系统** | 搭框架（管理器 + UI），不填具体成就内容 |
