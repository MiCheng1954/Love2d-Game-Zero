# REQUIREMENTS - Zero

> 记录最终确定下来的游戏需求，持续更新。

---

## 基本信息

- **游戏类型：** 幸存者类射击游戏（类 Vampire Survivors）
- **开发框架：** Love2D（Lua）
- **渲染方式：** 开发阶段使用代码绘制，后期替换资产

---

## 一、场景系统

- 游戏包含多个场景，每个场景有独特的特色和机制
- 场景类型多样，涵盖无限延伸、固定边界、滚动地图等

---

## 二、角色系统

- 多个可选角色，每个角色有：
  - 独特的主动技能
  - 独特的被动技能
  - 不同的基础属性
- 通用技能池：所有角色都可获取的技能，可在局内或局外获得
- **羁绊系统**：技能之间存在羁绊，激活后触发额外效果（类似自走棋）
- **传承技能**：每局结束时，根据本局 Build 选择一个传承技能，可携带至下一局

---

## 三、武器背包系统

- 武器背包为**可成长的二维网格**，可在战斗中扩展容量
- 背包扩展途径：击杀特定敌人掉落 / 升级奖励 / 地图探索
- 不同武器占据不同尺寸的格子（如 1×1、1×2、2×2 等）
- **武器融合**：N 把武器可合成新武器（如木棍 + 匕首 = 长枪），融合种类尽量丰富
- **相邻增益**：背包中相邻放置的武器互相提供增益效果
- **武器羁绊系统**：武器之间同样存在羁绊机制

---

## 四、升级系统

- 每次升级角色基础属性自动成长
- 升级奖励为多级选择结构：
  ```
  大类选择
  ├── 武器相关
  │   ├── 升级现有武器
  │   └── 获得新武器
  ├── 属性相关
  │   └── 各类属性加成
  └── 技能相关
      └── 获得/升级技能
  ```
- 部分角色升级时有**专属特殊选项**，需预留扩展接口
- 灵魂可用于刷新升级奖励选项

---

## 五、战斗系统

- 玩家自动攻击，专注移动闪避
- 有多种类型敌人：近战、远程、精英怪、Boss
- **Boss 刷新机制**：定时刷新 + 特殊条件触发
- 敌人拥有自己的技能和机制
- **重要架构原则**：玩家与敌人底层数值、技能系统共用同一套逻辑，统一抽象，不强行分离

---

## 六、游戏节奏（局内 20 分钟）

- **前 16 分钟**：8 次「慢 → 快 → 非常快 → 慢」循环（每次约 2 分钟）
- **后 4 分钟**：节奏持续不断加快，不再回落
- 快慢体现在敌人的**数量与质量**变化上
- 局长时间（20 分钟）需预留可配置接口，后续可调整

---

## 七、掉落与拾取系统

- 敌人掉落内容：
  - **经验值**：用于角色升级
  - **灵魂**：局内消耗货币（原"金币"）
  - **特殊事件触发器**：拾取后展开固定奖励，类型包括：
    - 属性包 / 武器包 / 技能包 / 背包成长 / 灵魂转换
  - 触发器奖励内容固定，玩家无法主动选择奖励类型
- 经验值与灵魂均为**自动吸附**拾取
- **吸附半径**作为玩家基础属性，可成长，经验与灵魂共用同一吸附半径

---

## 八、灵魂系统

- 怪物击杀掉落**灵魂**作为局内货币
- **局内用途**：
  - 刷新升级奖励选项
  - 触发器中的**灵魂转换**选项，消耗灵魂换取武器/属性/技能等奖励
- 局内灵魂**不带入局外**
- 局外灵魂系统后续单独设计

---

## 九、死亡与复活机制

- 死亡时触发传承技能结算界面，玩家二选一：
  - **选择传承技能** → 本局结束，携带传承技能进入下一局
  - **放弃传承** → 消耗复活机会，原地复活一次
- 复活机会：每局默认 1 次，可通过局外解锁获得更多次数，或局内特殊事件额外获取

---

## 十、视角与输入系统

- 游戏视角：俯视角（Top-down）
- 当前支持输入方式：WASD 键盘控制移动
- 需预留**输入系统抽象接口**，后续可扩展手柄 / 鼠标点击等方式

---

## 十一、UI / HUD 系统

- **局内常驻 HUD**（初版设计，后续可调整）：
  - 血量、经验条、等级、计时器、灵魂数量
- **背包界面**：TAB 键呼出 / 收起，不常驻显示

---

## 十二、玩家基础属性

| 属性 | 说明 |
|------|------|
| 最大生命值 | 血量上限 |
| 移动速度 | 玩家移动快慢 |
| 攻击力 | 基础伤害倍率 |
| 防御力 | 受到伤害减免 |
| 吸附半径 | 拾取经验/灵魂的范围，可成长 |
| 暴击率 | 触发暴击的概率 |
| 暴击伤害 | 暴击时的伤害倍率 |
| 攻击速度 | 武器攻击频率 |
| 经验加成 | 获得经验值的倍率 |
| 灵魂加成 | 获得灵魂数量的倍率 |
