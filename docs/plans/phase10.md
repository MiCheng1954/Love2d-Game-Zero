## 一、功能模块总览

### 1.1 结算统计界面
- 数据**逐条动画出现**（数字滚动/逐行跳出效果）
- 胜利与死亡使用**完全不同的视觉设计**（颜色/布局/标题均不同）
- 按任意键才能继续，**不自动跳转**
- 统计数据项：

| 统计项 | 来源 |
|--------|------|
| 存活时长 | `_rhythm:getElapsed()` |
| 最终等级 | `_player:getLevel()` |
| 总击杀数 | 新增 `_killCount` 计数器 |
| 收集灵魂总量 | `_player:getSouls()` |
| 激活的羁绊 | `bag._activeSynergies` |
| 击杀的Boss列表 | 新增 `_killedBosses` 列表 |
| 是否胜利 | `_victory` |

### 1.2 复活机制
- 每局初始 **1 次** 复活机会（`player._revives = 1`）
- 局内事件可追加复活次数（预留接口）
- **阵亡流程：**
  ```
  玩家阵亡
    ├── 还有复活次数 → 全屏暂停，弹出「复活 / 传承」二选一界面
    │     ├── 选「复活」→ 复活次数-1，清除周围一圈敌人，施加无敌buff，继续游戏
    │     └── 选「传承」→ 游戏以【失败】结束，进入传承技能选择界面
    └── 复活次数耗尽 → 游戏直接以【失败】结束，无传承选择，跳转死亡结算界面
  ```
- **胜利通关**：直接进入胜利结算界面，**不触发**传承选择
- 复活后清场半径、无敌时长等参数待实装时确定
- ⚠️ 无敌 buff 的系统性设计（Buff 管理器）放入 **Phase 10.1** 专项处理；本阶段用简化标记（`player._invincibleTimer`）临时实现

### 1.3 传承技能系统

#### 核心规则
- 每局只有 **1 个传承槽**，新一局的传承覆盖上一局（不累积）
- 本局是否有传承，**只取决于上一局是否触发了传承选择**
- 传承数据持久化至 `data/legacy.json`

#### 传承池分类
根据本局激活的羁绊类别，匹配对应传承池随机抽取3个候选：

| 传承大类 | 对应现有 Tag | 代表传承效果 |
|----------|-------------|-------------|
| 伤害 | 火力、精准、爆炸 | 攻击力+、暴击率+、暴击伤害+ |
| 科技 | 科技 | 技能CD缩短、弹速+、武器射速+ |
| 生存 | 防御 | 最大HP+、防御%+、复活次数+ |
| 爆发 | 爆发 | 技能CD-、主动技能增强 |
| 经济 | 辅助 | 经验获取+、拾取范围+、灵魂获取+ |

> 若本局无激活羁绊，则从**全类别混合池**中随机抽取。

#### 传承候选池（初版，各类别各 3~4 个）

| 传承 ID | 所属类别 | 效果 |
|---------|---------|------|
| `legacy_attack` | 伤害 | 攻击力永久 +8 |
| `legacy_crit_rate` | 伤害 | 暴击率永久 +5% |
| `legacy_crit_dmg` | 伤害 | 暴击伤害永久 +20% |
| `legacy_bullet_speed` | 科技 | 弹速永久 +40 |
| `legacy_cooldown` | 科技/爆发 | 所有技能 CD 缩短 10% |
| `legacy_attack_speed` | 科技 | 武器射速永久 +10% |
| `legacy_hp` | 生存 | 最大 HP 永久 +30 |
| `legacy_defense` | 生存 | 防御力永久 +3% |
| `legacy_speed` | 生存 | 移速永久 +15 |
| `legacy_exp` | 经济 | 经验获取永久 +20% |
| `legacy_pickup` | 经济 | 拾取范围永久 +40 |
| `legacy_souls` | 经济 | 灵魂获取永久 +20% |

#### 传承在游戏内的显示
- **背包技能面板**：传承技能和普通技能一起显示，但样式有区分（如金色角标「传承」）
- **战斗HUD**：技能槽右侧单独显示一个**圆形图标**，鼠标悬停或固定显示传承名称
- 传承作为**隐形被动**生效，不占技能槽

### 1.4 传承选择界面（`src/states/legacySelect.lua`）
- 展示 **3 张候选卡片**，左右居中排列
- 交互：**左右键**切换选中，**Enter** 确认
- 每张卡片显示：传承名称、所属类别、效果描述、代码绘制图标
- 确认后写入 `data/legacy.json`，跳转死亡结算界面

---

### 1.5 复活/传承二选一界面（`src/states/reviveUI.lua`）
- **全屏暂停**覆盖游戏画面，push 叠加
- 无倒计时，一直等待玩家操作
- 左右键选择「复活」/「传承」，Enter 确认
- 界面显示剩余复活次数、当前 HP 等关键信息

## 三、开发顺序

1. **`game.lua`** — 补充统计收集（`_killCount`/`_killedBosses`），死亡时走新流程
2. **`gameover.lua`** — 重写，胜利/死亡两套逐条动画结算界面
3. **`config/legacy.lua`** — 传承技能候选池 + 大类标注
4. **`src/systems/legacyManager.lua`** — 读写、池匹配、下局应用
5. **`src/states/reviveUI.lua`** — 复活/传承二选一界面
6. **`src/states/legacySelect.lua`** — 传承三选一界面
7. **`player.lua`** — `_revives` 字段、无敌临时实现、传承效果应用
8. **HUD 更新** — 技能槽右侧传承圆形图标
9. **i18n 补全**
