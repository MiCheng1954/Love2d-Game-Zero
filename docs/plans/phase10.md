# Phase 10 开发计划 — 死亡与传承

## 目标
实现传承技能结算流程、复活机制，完善结算界面，打通「死亡→传承→下一局」完整循环。

---

## 一、主要模块

### 1.1 传承技能系统
- 每局结束时（死亡或通关），根据本局 Build 生成 1~3 个传承技能候选
- 候选逻辑：从本局持有的技能/武器中提炼出对应的传承效果
- 玩家选择一个传承技能携带至下一局
- 传承技能存入 `data/legacy.json`（局外持久化）

### 1.2 死亡流程（二选一界面）
```
玩家死亡
  ├── 有复活机会 → 弹出选择界面
  │     ├── [选择传承技能] → 跳转传承选择界面 → 结算 → 主菜单
  │     └── [消耗复活机会] → 原地复活，继续游戏
  └── 无复活机会 → 直接跳转传承选择界面
```

### 1.3 复活机制
- 每局默认 1 次复活机会（`player._revives`）
- 复活时：HP 恢复 50%，短暂无敌帧（约2秒）
- 复活次数可通过局外解锁增加，或局内特殊事件获取

### 1.4 结算界面完善
- 显示本局统计：存活时长、击杀数、最高等级、收集的武器、激活的羁绊
- 显示选择的传承技能
- 按任意键/Enter 返回主菜单

### 1.5 传承技能下一局生效
- 游戏开始时读取 `data/legacy.json`
- 玩家以携带传承技能的状态开始新局
- 传承技能在技能槽中标注为"传承"
