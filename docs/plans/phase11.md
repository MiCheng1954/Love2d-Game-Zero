# Phase 11 开发计划 — HUD 与 UI 完善

## 目标
完善局内常驻 HUD、主菜单界面，实现触发器 UI，统一全游戏的 UI 视觉风格。

---

## 一、主要模块

### 1.1 局内常驻 HUD 完善
当前 HUD 为基础占位版，本阶段完善：
- **血量条**：视觉优化，显示当前/最大值，低血量变红闪烁
- **经验条**：显示当前经验/升级所需，等级数字
- **计时器**：局内游戏时长（倒计时或正计时，需确认）
- **灵魂数量**：图标+数字
- **武器快览**：当前背包中的武器列表（小图标，显示等级）
- **激活羁绊**：当前激活的武器/技能羁绊图标
- **复活次数**：心形图标显示剩余复活次数

### 1.2 触发器 UI（Trigger Pickup UI）
- 拾取「特殊事件触发器」后弹出奖励展示界面
- 触发器类型：属性包 / 武器包 / 技能包 / 背包成长 / 灵魂转换
- 奖励内容固定（不可选择），弹出后展示获得内容，3秒后自动关闭或按键确认
- 与升级选择界面视觉风格区分（升级=主动选择，触发器=固定奖励）

### 1.3 主菜单完善
- 标题 Logo（代码绘制风格）
- 菜单项：开始游戏 / 继续游戏（有存档时）/ 设置 / 退出
- 背景动画（简单粒子或几何图案）
- 角色选择入口（Phase 13 完善，此阶段预留）

### 1.4 UI 通用组件库
- 封装常用 UI 元素：按钮、进度条、面板、图标、浮窗
- 统一颜色规范和字体规范
- 便于后续各 Phase 复用
